## 页面布局建议

- **布局模式**: 上下分区（单页纵向滚动） + 悬浮侧边栏目录 —— 课件为线性学习流，学生按模块顺序推进，纵向滚动符合阅读习惯
- **视觉重心**: 每个模块的**交互演示区**（SVG/Canvas几何图形、滑块、拖拽区）为视觉重心，占模块 60%+ 空间；文字说明为辅
- **结果承载区**:
  - 答题类（课前小测、随堂练习、闯关游戏）：题目下方即时显示判分结果与解析
  - 操作类（折纸、拖拽排序）：操作区旁显示当前状态/成功提示/勋章解锁动画
  - 初始态：封面页为全屏欢迎态，其余模块默认展示初始几何图形与引导文案
- **响应式策略**:
  - 桌面端（横屏）：左右布局，左侧/右侧悬浮目录，主内容区居中（最大宽度 1200px）
  - 平板/手机（竖屏）：顶部悬浮目录按钮 + 进度条，内容全宽纵向排列，图形演示区自适应缩放

### 模块1：开篇导学·温故知新

- **页面目标**: 通过回顾旧知 + 实景类比 + 课前检测，建立"线线→线面→面面"的认知递进，引出二面角概念
- **功能点**:
  - **封面页交互**: 全屏封面，点击任意位置或"开始学习"按钮平滑滚动到下一模块，背景几何图形（CSS绘制的平面/直线装饰）有微动效
  - **垂直层级拖拽排序**: 三张卡片【线线垂直】【线面垂直】【面面垂直】可拖拽至三个目标框；提交后判断顺序，错误显示红色提示"先研究直线，再研究平面，请重新排序"，正确显示绿色总结"由线到面，定义→判定→性质"
  - **实景类比导入**: 四个 CSS/SVG 绘制的示意图标（水坝横截面、教室墙角、打开的书本、长方体），点击弹出标注框显示"棱""两个半平面"指向说明；底部可展开/收起"平面角 vs 二面角"对比表格
  - **课前小测**: 3 道单选题（每题 4 选项），逐题或整页提交，即时判分（正确绿色高亮，错误红色 + 显示解析），记录得分

---

### 模块2：新知1 二面角核心概念

- **页面目标**: 掌握二面角定义、平面角作法、直二面角概念，能在正方体/锥体中找到并计算简单二面角
- **功能点**:
  - **二面角定义动态演示**: SVG 绘制两个半平面（半透明填充）相交于棱 l，底部滑块控制开合角度（0°~180°），实时显示角度数值；标注"棱""半平面α""半平面β"；点击"三种记法"按钮弹出 α-l-β、α-AB-β、P-l-Q 三种记法说明
  - **平面角分步动画**: 点击"下一步"分步演示：①棱上取点 O → ②在 α 内作 OA⊥l → ③在 β 内作 OB⊥l → ④形成∠AOB（平面角）；顶点 O 可左右拖动，角度数值实时不变，弹出结论"平面角大小与棱上顶点位置无关"
  - **角度分类滑动条**: 滑块拖动 0°~180°，途经区间自动标注"锐二面角（0°<θ<90°）""直二面角（θ=90°）""钝二面角（90°<θ<180°）""平二面角（θ=180°）"；90° 时特殊高亮，弹出"直二面角 = 两平面互相垂直"
  - **易错点提示卡片**: 点击展开，展示"两边必须都垂直于棱"的反例（一边不垂直的错误画法），对比正确画法
  - **教室实景填空题**: SVG 绘制教室墙角示意图（墙面+地面），两个填空框："棱是____""二面角度数是____"，输入后自动批改，正确变绿错误变红
  - **例题交互（例1 正方体）**: SVG 绘制正方体 ABCD-A₁B₁C₁D₁，点击棱 BC 高亮，平面角 ∠ABA₁ 逐步高亮显示；点击"显示推导"逐行展示证明步骤，最终答案 45°
  - **例题交互（例2 正方形+垂线）**: SVG 绘制正方形 ABCD 及 PA⊥平面ABCD，两小问分步动画（①求平面PAB与平面ABCD的二面角→90°；②求平面PBC与平面ABCD的二面角→45°），每步点击展开
  - **解题口诀滑动卡片**: 三张横向滑动卡片"一作（作平面角）""二证（证明是平面角）""三求（计算角度）"，左右滑动切换，每张含口诀+简要说明

---

### 模块3：新知2 面面垂直判定定理

- **页面目标**: 理解并掌握面面垂直的判定定理，能用三种语言表述，能在四面体中完成证明
- **功能点**:
  - **长方体探究模型**: CSS 3D / SVG 透视绘制长方体，高亮一个面内的一条棱（垂直于底面），演示"一个平面过另一个平面的一条垂线 → 这两个平面垂直"，配动画高亮两个垂直平面
  - **三语言切换**: 三个切换按钮【文字语言】【图形语言】【符号语言】，点击切换展示：
    - 文字语言："如果一个平面过另一个平面的垂线，那么这两个平面垂直"
    - 图形语言：SVG 示意图（α、β 两个平面 + 垂线 a）
    - 符号语言：`a⊥α，a⊂β ⇒ α⊥β`
    - 底部简记："线面垂直 ⇒ 面面垂直"
  - **定理分步证明**: 点击"下一步"按钮逐行显示证明过程（已知、求证、证明步骤），当前行高亮
  - **四面体例题（教材例7）**: SVG 绘制四面体 ABCS，题干"AB=BC=CA=SB=SC=SA，求证平面ABC⊥平面SBC"；点击"作辅助线"一键生成 SD⊥BC、AD⊥BC；证明步骤以填空形式呈现（挖空关键词，如"线面垂直""SSS"等），下拉选择/填空后自动批改
  - **证明流程图**: 横向四步流程图"分析垂直关系 → 找平面垂线 → 证线面垂直 → 套用判定定理"，每步可点击展开说明

---

### 模块4：虚拟折纸实操

- **页面目标**: 通过动手折纸操作，直观感受二面角变化，强化直二面角概念
- **功能点**:
  - **虚拟折纸交互**: Canvas/SVG 绘制长方形纸张，底部滑块控制折叠角度（0°~180°），纸张沿中线动态弯折，实时显示二面角度数
  - **任务解锁机制**: 设置两个目标角度：
    - 任务1：折出 45°（±2° 误差内）→ 提示"接近了/精准！"
    - 任务2：折出 90°（±2° 误差内）→ 提示"直二面角！两平面垂直！"
    - 两个任务都完成 → 解锁"作图大师"勋章，弹出勋章动画
  - **生活实景轮播**: 三个 CSS/SVG 示意图标（屋顶坡面、文件夹、阶梯水坝），左右切换，每个标注二面角的实际应用，点击可放大查看

---

### 模块5：分层随堂练习

- **页面目标**: 三层难度递进练习，即时批改，巩固所学
- **功能点**:
  - **层级切换 Tab**: 三个 Tab【基础层】【提升层】【拓展层】，点击切换
  - **基础层（3题）**: 2道单选题 + 1道填空题
    - 题1：二面角的记法判断
    - 题2：直二面角的识别（给出4个图形选正确的）
    - 题3：正方体中二面角度数填空
    - 全部自动批改，错题标红 + 显示解析
  - **提升层**:
    - **手绘平面角**: Canvas 画布上给出一个二面角示意图，用户用鼠标/触屏手绘出平面角，提交后系统判断（检测是否两边都垂直于棱、顶点在棱上），给出反馈
    - **证明步骤填空**: 给出一道证明题，步骤中挖空 3-4 处关键词，下拉选择补全，自动批改
  - **拓展层**: 一道综合大题（题干 + 图形），点击"分步解析"逐步展开提示/解答，不强制判分，供学有余力的学生自学
  - **错题记录**: 所有答错的题自动记入错题本（localStorage），模块底部显示"本次错题 X 道"，可展开查看错题列表

---

### 模块6：课堂小结+课后任务

- **页面目标**: 梳理知识体系，通过闯关游戏检验掌握程度，布置分层作业
- **功能点**:
  - **知识思维导图**: 可点击展开/收起的树形结构：
    - 二面角 → 定义 → 记法（3种）
    - 二面角 → 平面角 → 作法（一作二证三求）→ 分类（锐/直/钝/平）
    - 直二面角 → 面面垂直定义
    - 面面垂直 → 判定定理 → 三语言 → 应用
    - 点击节点高亮对应内容，可展开/收起子节点
  - **闯关小游戏（5道判断题）**:
    - 逐题弹出判断题（如"两个平面垂直，则一个平面内的任意直线都垂直于另一个平面"），选择"正确/错误"
    - 答对前进一格，答错后退/提示
    - 5 题全部答对 → 解锁"证明能手"勋章，弹出庆祝动画
    - 答错可查看解析后重试
  - **分层课后任务（三选一）**:
    - 基础任务：完成教材习题 X 道（列出题号）
    - 实践任务1：折纸拍照——折出 60° 二面角并拍照记录
    - 实践任务2：实物拍摄——寻找生活中的面面垂直实例并拍照
    - 每个任务有"已完成"勾选框（仅本地记录）
  - **课堂结束页**: 总结语 + 进度条 100% + 已获得勋章展示 + "重新学习"按钮

## 技术选型备注

- **几何图形渲染**: 以 SVG 为主（适合交互标注、分步动画、响应式缩放），Canvas 用于手绘题和折纸动态效果
- **不依赖外部图片**: 所有示意图、实景图标均用 CSS/SVG/Canvas 原生绘制，保证离线可运行
- **数学符号**: 使用 Unicode 数学字符 + 必要的内联 SVG 绘制复杂公式，或使用 MathML / 简单 CSS 排版，不引入额外数学公式库（保持轻量）

- **参考意图**: Free Direction —— 参考链接不可直接读取，按用户给出的详细模块结构与教学语义自主建立视觉系统

- **独特记忆点**: 以"二面角开合"为视觉母题——封面背景、章节过渡、滑块控件、勋章边框均复用"两个半平面相交"的几何形态，让抽象概念成为贯穿全课的视觉钩子

## 2. Art Direction

- **方向名**: 几何教学极简风
- **Design Style**: Swiss Minimalist + 轻量几何装饰 —— 数学教学需要秩序与精确，瑞士极简的网格排版保证公式与图形清晰可读；局部几何装饰呼应立体几何主题，避免枯燥
- **DNA 参数**: 圆角 subtle（rounded-md） / 阴影 subtle（shadow-sm） / 间距 standard（gap-4 / p-6） / 字体方向 无衬线正文 + 几何感标题 / 装饰手法 半透明平面叠加 + 细线棱边 + 角度刻度
- **应用类型**: Interactive Lesson —— 纵向滚动 + 锚点跳转，每模块为一个独立学习节

## 3. Color System

**色彩关系**: 深蓝主色（知识/理性）+ 橙色辅助（交互/重点）+ 纸白背景 + 灰蓝边框，成功绿与错误红饱和度与主色对齐
**配色设计理由**: 深蓝 #165DFF 承担主按钮、当前章节、关键定理高亮，传递数学的严谨与可信；橙色 #FF7D00 用于交互提示、重点标注、勋章，形成视觉跳脱；中性色采用偏冷灰，与主色统一色温；语义色（对/错）控制饱和度，不压过教学内容
**主色推导**: 用户指定 #165DFF 为主色，对应 hsl(221 100% 52%)；accent 与 border 从主色降饱和降明度衍生；橙色 #FF7D00 作为第二强调色，仅占 <5% 视觉面积
**使用比例**: 65% 中性（bg/card/text/border）/ 25% 辅助浅蓝（accent/muted）/ 8% primary / 2% accent-orange；primary 只用于 CTA、当前章节指示、定理标题；橙色只用于重点标注与勋章

| bg | `--background` | `bg-background` | hsl(214 35% 97%) | 冷调浅蓝灰底，接近打印纸质感 |

- 错误（答错/警示）: bg hsl(0 90% 96%) / border hsl(0 85% 80%) / text hsl(0 80% 45%) —— #F53F3F 降饱和至与 primary 对齐
- 重点（橙色标注）: bg hsl(28 100% 95%) / border hsl(28 95% 75%) / text hsl(28 100% 45%) —— #FF7D00 仅用于勋章、重点提示、拖拽热区

## 4. 字体与节奏

- **font-display**: Noto Sans SC —— 中文几何无衬线，笔画均匀，适合数学标题与定理陈述
- **font-body**: Noto Sans SC —— 正文清晰易读，长时间学习不疲劳；公式配合系统 serif 字体
- **字号**: H1 text-4xl md:text-5xl；H2 text-2xl md:text-3xl；H3 text-xl；body text-base；muted text-sm；公式/角标 text-sm
- **圆角**: 中（rounded-md）—— 卡片与按钮 8px，输入框 6px，徽章全圆；几何图形保持锐利棱边

## 5. 全局布局契约

- **Reference Layout Use**: 按需求结构推导，6 大模块纵向排列，每模块为一个 section，锚点跳转
- **Page / Section Order**: 开篇导学 → 二面角概念 → 面面垂直判定 → 虚拟折纸 → 随堂练习 → 课堂小结，与需求 1:1 对齐
- **Standard Content Zone**: max-w-3xl md:max-w-4xl lg:max-w-5xl + `mx-auto` —— 教学内容以阅读为主，控制行长，几何演示区可局部突破
- **Shell / Frame Alignment**: 内容区与悬浮侧边栏独立滚动；顶部进度条全宽，内容区居中
- **Padding & Rhythm**: `px-4 md:px-6 py-10 md:py-14`，模块间距 `py-16 md:py-20`，保持 8px 倍数
- **Full-bleed Zones**: 封面页背景几何装饰、模块过渡分隔条、SVG/Canvas 演示区可全宽；文字与按钮始终受内容区约束
- **Local Narrowing**: 填空题、选择题、证明步骤等文字密集区收窄至 `max-w-2xl mx-auto`
- **Overflow Strategy**: 长公式、证明流程、手绘 canvas 使用 `overflow-x-auto` 或固定高度容器
- **Flexibility Boundary**: 允许移动端卡片 padding 减至 p-4、字号降一级；不允许切换主色、圆角系统或模块顺序

## 6. 视觉与动效

- **装饰**: 半透明平面叠加 + 细线棱边 + 角度刻度弧
- **阴影/边界**: 轻 —— 卡片 shadow-sm，聚焦态 ring-2 ring-primary/30；几何图形用 border 而非阴影表达立体感
- **动效**: 精致克制 —— 滑块拖动实时反馈（100ms），分步演示淡入上移（250ms ease-out），答对/答错轻微缩放+颜色变化（200ms），页面切换平滑滚动（300ms）；不使用夸张弹跳或旋转

## 7. 组件原则

- 按钮：主按钮 solid primary，次按钮 outline，幽灵按钮用 accent 底；所有按钮有 focus-visible ring
- 拖拽卡片：默认 shadow-sm + border，拖拽中 scale-105 + shadow-md + border-primary
- 滑块：轨道浅灰，进度条 primary，thumb 圆形带橙色内环（呼应二面角视觉母题）
- 答题反馈：正确绿边+绿底轻填充+对勾图标；错误红边+红底轻填充+叉号+解析展开
- 勋章：圆形徽章，橙色描边，内部几何图形，解锁时缩放+光晕动画
- 几何演示区：白底 + 1px border + 轻微内边距，SVG 图形线条深蓝，高亮橙色，辅助线虚线灰蓝

## 8. Image Direction

- **Image Role**: 无强制图片需求 —— 所有几何图形、实景示意（水坝/墙角/书本/长方体/正方体/四面体）均用 CSS/SVG/Canvas 原生绘制，保证离线可运行与教学精确性
- **Image Art Direction**: 无强制图片需求；封面与章节过渡的几何装饰由 CSS 渐变 + SVG 多边形生成，风格统一为"线框半平面 + 淡色填充 + 棱边高亮"
- **Image Prompt Keywords**: 无
- **Image Avoidance**: 避免使用外部图片素材、通用教学插图、人物照片、3D 渲染图；所有视觉元素必须代码生成，确保数学精度与离线可用

## 9. Anti-patterns

- **Split personality**: 各模块各自为政，有的用卡片有的无边框；全站统一 card/border/shadow 语言
- **Geometry chaos**: 几何图形配色随意，棱、面、角、垂线没有统一颜色编码；建立图形图例：棱=深灰实线，半平面=淡蓝半透明，平面角=橙色弧线，垂线=绿色虚线
- **Over-orange**: 橙色铺满按钮、标题、图标，压过主色；橙色仅限重点标注、勋章、拖拽热区，占比 <5%
- **Invisible interaction**: 滑块、拖拽、点击图形只有 hover 没有 focus-visible；所有可交互元素必须有键盘可达与可见焦点态
- **Mono-hue tyranny**: primary 同时用于按钮、tab、图标、边框、链接、标题；按 65-25-8-2 比例分配，标题用 text-foreground，图标用 text-muted，链接用 text-primary underline
- **Status color drift**: 成功绿与错误红饱和度过高，像报警灯；语义色饱和度与 primary 对齐 ±15%，用浅底+边框而非纯色块表达状态
